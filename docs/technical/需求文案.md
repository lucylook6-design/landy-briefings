# 需求文案
HR 智能体需求分析0327——会议需求讨论成果纪要

主题： AI驱动的简历筛选的技术需求分析

一、核心需求概述
聚焦解决IRPO招聘中，HR 管理工作中匹配精度低、简历筛选重复低效、JD撰写不规范、招聘合规等痛点，优先服务公司自身人力U盾中小企业RPO引流业务，后续拓展对外商业化。

二、技术路线分析
（一）AI简历筛选的核心技术方案
￼简历结构化切片处理：先将不同格式（Word、PDF等）的简历按项目经历、工作经历、基本信息等模块切片，整理成标准结构化模板。
￼正反翻译匹配逻辑：一方面从简历反生成对应JD，另一方面对照JD要求匹配候选人，再用embedding向量化做一轮基于语义的初筛，得到统一标准的匹配分数。
￼初筛后二次精准匹配：初筛后仅保留5-8份候选人，再用大模型做二次评判，给出排序和推荐理由，最终供猎头挑选3份推荐给甲方。
￼成本控制效果：该方案token消耗仅为全流程大模型处理的1/50，若从200份简历中筛选，token消耗仅为常规方案的1%，成本降低效果明显。
（二）多招聘平台简历获取方案
￼有API的平台处理方式：头部招聘平台（BOSS直聘、猎聘、拉钩等）开放合作后有官方API，可直接对接获取简历，工作量和成本都更低。
￼无API的平台处理方式：采用基于自动化测试工具改造的AI浏览器，在沙箱环境下通过分析网页结构底层操作模拟人工浏览，自动判断是否需要付费下载简历，可正常完成简历获取。(技术难点，效果取决于招聘平台对爬取的对抗)
￼多账号管理方案：支持按不同客户、不同平台分类管理账号密码，可配置自动化轮换登录流程，适配猎头/RPO机构对接多个甲方客户的使用场景。中文场景优先选用国内大模型，出海场景选用海外模型。

三、用户场景与产品实现要求
1. 第一阶段做轻量化工具，考虑小程序/pc作为前端入口，对接企业微信自动交付结果，保留人工后端对接，保留服务温度；支持多客户多招聘需求管理，支持AI辅助优化不规范JD、对齐用人需求
2. 当前核心AI逻辑已验证，本次重点开发前端UI与交互
3. 商业模式：第一阶段做标准化工具提效，第二阶段沉淀垂直简历库，打造垂直细分招聘平台

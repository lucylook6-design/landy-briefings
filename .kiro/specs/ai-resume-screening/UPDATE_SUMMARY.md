## 📊 核心变更说明

### 权限模型变更

**之前**: 用户直接访问客户  
**现在**: 用户通过 USER_COMPANY 关联多个公司

```
USER ←→ USER_COMPANY ←→ CLIENT
         (多对多)
```

### 平台对接策略

**MVP 阶段**: 仅支持 BOSS 直聘  
**v1.1 阶段**: 扩展支持猎聘、拉钩等其他平台

### 数据隔离机制

- 数据库行级安全策略（RLS）
- 用户切换公司后，所有数据自动按当前公司过滤
- 严格防止跨公司数据访问

## 🎯 新增功能特性

### 1. 用户-公司权限管理 (F-019)
- 一个用户可以管理多个公司
- 支持公司切换（顶栏下拉选择）
- 显示用户在每个公司的角色（管理员/成员）
- 所有数据按公司严格隔离

### 2. BOSS直聘账号绑定 (F-020)
- 每个公司可以绑定一个BOSS直聘账号
- 账号密码使用 AES-256 加密存储
- 支持测试账号连接状态
- 显示账号最后使用时间和状态
- 支持修改账号信息

## 🔐 安全特性

### 密码加密
- 算法: AES-256
- 存储: encrypted_password 字段
- 访问控制: 仅授权管理员可修改

### 数据隔离
- 机制: 数据库行级安全策略（RLS）
- 验证: 所有查询必须通过 RLS 校验
- 测试: 隔离测试用例（T-019）

## ✅ 新的工作方式（必读）

### 核心变化

**以前**：修改 `index.html` 添加简报数据  
**现在**：修改 `reports-data.json` 添加简报数据

**原因**：
- `index.html` 包含页面结构和数据，Kiro和你都修改 → 冲突
- 现在分离：Kiro维护结构，你维护数据 → 各自独立文件 → 无冲突

## 📋 每日简报更新流程（新版）

### 步骤1：拉取最新代码
```bash
cd ~/briefings  # 或你的工作目录
git pull origin main --rebase
```

### 步骤2：创建简报HTML文件
在 `reports/` 目录创建新文件，例如：
```
reports/2026-04-29.html
```

### 步骤3：更新 reports-data.json（重要！）

**打开文件**：`reports-data.json`

**在数组开头添加新记录**：
```json
[
  {
    "date": "2026-04-29",
    "title": "今日简报",
    "desc": "最新AI、液冷、资本市场动态",
    "tags": ["ai", "stock", "energy"],
    "file": "reports/2026-04-29.html"
  },
  {
    "date": "2026-04-28",
    ...现有记录...
  }
]
```

**注意**：
- 在数组**开头**添加（最新的在最前面）
- 保持JSON格式正确（逗号、引号、括号）
- 不要删除旧记录

### 步骤4：提交并推送
```bash
# 再次拉取确保最新
git pull origin main --rebase

# 添加文件
git add reports/2026-04-29.html reports-data.json

# 提交
git commit -m "简报归档: 2026-04-29"

# 推送
git push origin main
```

## 📝 reports-data.json 格式说明

### 完整示例
```json
[
  {
    "date": "2026-04-29",
    "title": "今日简报",
    "desc": "最新AI、液冷、资本市场动态",
    "tags": ["ai", "stock", "energy"],
    "file": "reports/2026-04-29.html"
  }
]
```

### 字段说明
- `date`: 日期，格式 `YYYY-MM-DD`
- `title`: 标题
- `desc`: 描述（可以包含HTML标签如 `<strong>`）
- `tags`: 标签数组，可用值：`"ai"`, `"stock"`, `"energy"`, `"robot"`
- `file`: 文件路径，格式 `reports/YYYY-MM-DD.html`

## 🔧 如果遇到冲突

### 场景：git pull 时提示冲突

```bash
# 1. 查看冲突文件
git status

# 2. 如果是 reports-data.json 冲突
# 打开文件，找到冲突标记：
# <<<<<<< HEAD
# 你的改动
# =======
# 远程的改动
# >>>>>>> origin/main

# 3. 手动合并（保留双方的记录）
# 删除冲突标记，合并数组内容

# 4. 标记为已解决
git add reports-data.json

# 5. 继续rebase
git rebase --continue

# 6. 推送
git push origin main
```

### 紧急情况：不确定如何处理

```bash
# 1. 取消当前操作
git rebase --abort

# 2. 保存你的改动到临时文件
# 复制 reports/2026-04-XX.html 和你在 reports-data.json 中添加的记录

# 3. 重置到远程版本
git reset --hard origin/main

# 4. 重新应用你的改动
# 重新创建简报文件和更新 reports-data.json

# 5. 重新提交
git add reports/2026-04-XX.html reports-data.json
git commit -m "简报归档: 2026-04-XX"
git push origin main
```

## ❓ 常见问题

**A**:

### Q3: reports-data.json 的格式我不确定对不对？
**A**: 可以用在线JSON验证工具检查，或者推送前让Kiro看一眼。

### Q4: 我需要修改旧简报的内容怎么办？
**A**: 直接修改 `reports/2026-04-XX.html` 文件，然后正常提交推送即可。
